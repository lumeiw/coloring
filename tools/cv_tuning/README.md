# CV-тюнинг

Отладочный инструмент для подбора параметров распознавания областей. Это
самостоятельная Python-реплика пайплайна из
[`lib/features/coloring/data/local_cv_data_source.dart`](../../lib/features/coloring/data/local_cv_data_source.dart):
прогоняет картинку раскраски и рисует, какие участки скрипт распознал, а какие
«не воспринял» (мёртвые зоны) — не пересобирая всё Flutter-приложение.

Значения в Dart-коде приложения соответствуют пресету **`new2400`**.

## Установка (один раз)

```bash
cd tools/cv_tuning
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

## Запуск

```bash
.venv/bin/python cv_pipeline.py <картинка.png> <префикс_вывода> [old|new|new2400]
```

- **картинка** — PNG или JPG раскраски (PDF сначала конвертни в PNG, напр.
  `sips -s format png in.pdf --out out.png`);
- **префикс** — начало имён выходных файлов;
- **пресет** — `old` (текущий код до правок), `new`, `new2400` (то, что сейчас в приложении).

На выходе:

- `<префикс>_<пресет>_regions.png` — карта участков: каждый распознанный участок
  своим цветом, **чёрное** — линии/фон, **красное** — мёртвые зоны (внутри
  контура, но участок не распознан → закрасить нельзя);
- `<префикс>_<пресет>_overlay.png` — та же карта поверх исходной картинки;
- в консоль — статистика (`dead_pct` — % нераспознанного, `dropped` — сколько
  участков выброшено как шум).

## Как переносить найденные параметры в приложение

Подобрал хорошие значения — перенеси константы в начало
`local_cv_data_source.dart` (`_maxDim`, `_minAreaPx`, `_holeFillPx`) и,
при необходимости, параметры `adaptiveThreshold` / `MORPH_CLOSE` по месту.
Уже импортированные работы CV не пересчитывает (результат кешируется в БД) —
чтобы увидеть новую нарезку, импортируй картинку заново.
