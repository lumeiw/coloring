# Раскраска по номерам (Coloring)

Офлайн-приложение «раскраска по номерам» на **Flutter**. Импортирует PDF или фото
чёрно-белой раскраски, локально (на устройстве) распознаёт замкнутые области с
помощью компьютерного зрения и даёт закрашивать их пальцем — краска не выходит за
контуры. Никакой сети: все вычисления и хранение — на устройстве.

> Целевые платформы: **iOS / iPad**. Material 3, светлая тема, пастельная палитра,
> шрифты Nunito Sans + JetBrains Mono.

## Возможности

- **Импорт** раскраски из PDF (выбор файла и страницы) или из **фото** галереи, плюс
  встроенный демо-образец.
- **Компьютерное зрение офлайн** (OpenCV в изоляте): grayscale → CLAHE → unsharp mask
  (display-версия) → медианный денойз → adaptive threshold → морфологическое замыкание
  контуров → `connectedComponents` → заливка дырок (цифры внутри областей) → карта
  регионов. Результат кешируется в SQLite, повторное открытие мгновенное.
- **Движок раскрашивания**: `CustomPainter` поверх свободной трансформации холста
  (сдвиг/зум до 1200%/поворот двумя пальцами), клиппинг мазка по маске региона через
  `saveLayer` + `BlendMode.dstIn`, кисть с настраиваемым размером, пипетка, подсветка
  активной области, undo/redo.
- **Два режима**: красить только внутри контуров (по умолчанию) или где угодно на
  рисунке — чтобы нивелировать неточности распознавания.
- **Галерея** «Мои работы» с прогрессом и превью; прогресс и лог мазков сохраняются в
  базе, работа восстанавливается при повторном открытии.

## Стек и архитектура

Clean Architecture, послойно на каждую фичу: `presentation / domain / data`.

| Область | Технология |
| --- | --- |
| Состояние | `flutter_bloc` (cubit) |
| Навигация | `go_router` |
| DI | `injectable` + `get_it` |
| Локальная БД | `drift` (SQLite) + `path_provider` |
| Модели/юнионы | `freezed` + `json_serializable` |
| Растеризация PDF | `pdfrx` (pdfium) |
| Компьютерное зрение | `opencv_dart` (dartcv) + `dart:ui` |
| Выбор файла/фото | `file_picker` |
| Кодогенерация | `build_runner` |

> `dio` присутствует в зависимостях, но не используется — задел на опциональную
> синхронизацию в будущем (контракт `CvRepository` спроектирован так, чтобы можно было
> добавить `RemoteCvDataSource`, не меняя domain).

```
lib/
├── app/            # MaterialApp.router, go_router
├── core/           # DI, тема (токены M3), drift-база, общие домен-модели
└── features/
    ├── import/     # выбор PDF/фото, растеризация, оркестрация импорта
    ├── coloring/   # CV-пайплайн, движок рисования, cubit, хранилище
    └── gallery/    # список работ поверх потока drift
```

## Запуск локально

Требуется Flutter (проект собран на **3.41.4**, Dart 3.11) и Xcode.

```bash
flutter pub get
dart run build_runner build          # кодогенерация (drift/freezed/injectable)
flutter run -d <ios-device-or-simulator>
```

Для установки на реальное устройство настройте подпись в Xcode
(`ios/Runner.xcworkspace` → Signing & Capabilities → Team).

## CI: сборка IPA

В [`.github/workflows/build-ipa.yml`](.github/workflows/build-ipa.yml) настроена
сборка **неподписанного** IPA на macOS-раннере (по push в `main` и вручную через
_Run workflow_). Готовый `.ipa` публикуется в **[Releases](../../releases)** (и
дополнительно доступен как артефакт запуска).

Неподписанный IPA нельзя поставить напрямую через App Store/Xcode — он подходит для
сторонней подписи/сайдлоада (AltStore, Sideloadly и т.п.). Чтобы собирать
**подписанный** IPA, добавьте секреты с сертификатом и provisioning-профилем и
включите codesign (см. комментарии в workflow).
